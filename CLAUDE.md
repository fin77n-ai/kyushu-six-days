# 九州六日旅行攻略

## 项目结构

- `index.html`：单页旅行攻略，包含样式、行程内容与页面交互。
- `manifest.webmanifest`：PWA 元数据与图标配置。
- `sw.js`：离线缓存文件列表与 Service Worker 逻辑。
- `icon.svg`、`apple-touch-icon.png`、`icon-192.png`、`icon-512.png`：站点与主屏幕图标。

## 修改规则

- 保留现有单页结构、移动优先布局和轻松实用的中文语气。
- 实测信息应合并到对应日期和专题章节，同时保留集中版避坑摘要。
- 交通信息必须同时说明方向、站点/站台与现场辨认方法；Google Maps 仅作为辅助。
- Plan B 是替换路线，不是叠加活动；雨天只保留一个核心地点。
- 行李规模固定为一大一小。
- 继续使用原仓库和 GitHub Pages 基础地址，不更换站点 URL。
- 修改时按 `id="dN"` 日程边界定位，避免内容插入错误日期。

## 验收

- 检查 HTML、manifest 与 Service Worker 语法及资源引用。
- 使用 390×844 手机视口检查无横向溢出、文字不遮挡、锚点和交互可用。
- 核对关键外链、地图定位和浏览器控制台错误。
